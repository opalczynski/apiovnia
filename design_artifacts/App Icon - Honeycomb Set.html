Trzy warianty rysunku, wybór wg rozmiaru

Aby ikona była ostra na każdym rozmiarze, nie skaluję jednego SVG do wszystkiego — używam trzech wariantów. Outline-y poniżej 1px znikają lub się sklejają w plamę, dlatego <=24px to wypełniony hex.

  • full≥ 64 px
  • + outer ring≥ 128 px
  • + drop glint≥ 256 px
  • mid32 – 48 px
  • small (filled)≤ 24 px
  • bg gradient≥ 128 px

Build command Tauri zrobi .ico automatycznie z PNG. Ręcznie:
magick convert 16.png 24.png 32.png 48.png 64.png 128.png 256.png apiovnia.ico
Squircle vs. square Windows 11 taskbar nie maskuje — ikony zostają takie jakie są w pliku. Squircle wygląda spójnie z macOS, ale system Win-natywnie używa kwadratu. Squircle z gradientem działa OK, tylko Start menu doda do niego cień.

Install path Wszystko ląduje w /usr/share/icons/hicolor/<size>/apps/. Po instalacji odpal gtk-update-icon-cache.
.desktop entry W /usr/share/applications/apiovnia.desktop wpis Icon=apiovnia — bez ścieżki, bez rozszerzenia. Theme engine sam dobierze rozmiar.
Tauri · tauri.conf.json
single source · bundler genuje pozostałe targety automatycznie
cargo tauri build cross-platform
Wrzuć źródłówki do src-tauri/icons/ i wpisz je do configu. Tauri sam wygeneruje .icns / .ico z PNG-ów (potrzebuje min. 32×32, 128×128, 128×128@2x = 256×256, plus icon.png 512+ i icon.icns/.ico).
// src-tauri/tauri.conf.json { "tauri": { "bundle": { "identifier": "dev.apiovnia.app", "icon": [ "icons/32x32.png", // linux, win small "icons/128x128.png", // linux mid "icons/128x128@2x.png", // = 256, macos retina src "icons/icon.icns", // macos final "icons/icon.ico", // windows final "icons/icon.png" // fallback, 512×512 master ] } } }
Minimum required Tauri zbuduje cross-platform jeśli masz: 32×32.png, 128×128.png, 128×128@2x.png, icon.icns, icon.ico. Resztę (drobne rozmiary) generuje bundler.
Helper command Z jednego masterowego 1024×1024:
cargo tauri icon ./src-tauri/icons/apiovnia.png
— wygeneruje wszystkie wymagane warianty automatycznie.
